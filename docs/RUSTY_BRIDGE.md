# Rusty Bridge

Rusty is the future backend bridge placeholder for VolumeGuardian.

## Meaning

Rusty exists as the clean connection point between the current PowerShell-based VolumeGuardian scripts and any future compiled backend.

## V1 Rule

Rusty does not take control.

Rusty does not modify system volume.

Rusty only confirms that the project has a safe future path for backend connection work.

## Future Direction

If Rusty is expanded later, it should stay focused on VolumeGuardian only:

- Read current state
- Report lock status
- Help manage one safe guardian process
- Keep logs clean
- Respect unlock control

## Principle

Rusty connects old code to future code without turning this project into a giant unrelated system.
